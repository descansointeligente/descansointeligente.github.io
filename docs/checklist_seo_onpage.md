# Checklist SEO On-Page (Antes de Publicar)

Usa esta lista de verificación para asegurar que cada nueva página o artículo esté perfectamente optimizado antes de hacer "commit".

## 1. Palabras Clave y Meta Etiquetas
- [ ] **Title Tag**: Incluye la Keyword Principal al inicio | Marca. (Máx 60 caracteres).
- [ ] **Meta Description**: Incluye Keyword + CTA + Beneficio. (Máx 155 caracteres).
- [ ] **URL (Slug)**: Corta, con guiones, sin "stop words" (`/mejor-cojin-coxis/` vs `/cual-es-el-mejor-cojin-para-el-coxis/`).
- [ ] **H1 Único**: Solo un H1 por página, idéntico o muy similar al Title.

## 2. Contenido y Estructura
- [ ] **Jerarquía H2/H3**: Estructura lógica. H2 para secciones principales, H3 para detalles.
- [ ] **Keyword Density**: Natural. Usar sinónimos y términos LSI (Latent Semantic Indexing).
- [ ] **Primer Párrafo**: La Keyword principal aparece en las primeras 100 palabras.
- [ ] **Legibilidad**: Párrafos cortos, uso de listas (ul/ol), negritas para destacar ideas.

## 3. Elementos Visuales
- [ ] **Imágenes**:
    *   Formato moderno (`.webp`).
    *   Nombre de archivo descriptivo (`cojin-ergonomico-oficina.webp`).
    *   Atributo `alt` descriptivo y con keyword si procede.
    *   Dimensiones correctas (no subir una foto de 4000px para mostrarla a 300px).
- [ ] **Video**: Embed de YouTube si aporta valor (aumenta tiempo en página).

## 4. Enlazado Interno (Interlinking)
- [ ] **Enlaces Salientes**: Al menos 1 enlace a otro artículo relacionado de nuestra web (Cluster).
- [ ] **Anchor Text**: Variado y descriptivo (no usar siempre "haz clic aquí" ni siempre la keyword exacta).
- [ ] **Breadcrumbs**: Verificar que las migas de pan son lógicas.

## 5. Conversión (CRO) y Afiliación
- [ ] **Disclaimer**: Aviso de afiliados visible ("En calidad de Afiliado de Amazon...").
- [ ] **Enlaces de Afiliado**:
    *   Atributo `rel="nofollow sponsored"`.
    *   Se abren en pestaña nueva (`target="_blank"`).
    *   Están etiquetados correctamente en Amazon Associates (IDs de seguimiento correctos).
- [ ] **Botones**: Funcionan y llevan al producto correcto.

## 6. Técnico
- [ ] **Datos Estructurados**: Validar Schema (Article, Product, FAQPage) con Rich Results Test.
- [ ] **Mobile Friendly**: Verificar visualización en móvil (Chrome DevTools).
- [ ] **Velocidad**: Verificar que no carga scripts innecesarios.
